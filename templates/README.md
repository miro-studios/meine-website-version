# Templates

Interne Basis-Templates für Kundenprojekte. Nicht öffentlich, kein Routing.

## Verfügbare Templates

| Template | Use-Case |
|----------|----------|
| `one-pager-base/` | Single-Page-Sites (Lead-Capture, Product-Launch) |
| `multi-page-base/` | Mehrseitige Sites (Service-Anbieter, Restaurants, Studios) |

## Workflow für Kundenprojekte

1. Template-Ordner kopieren in neues Repo
2. `site.config.ts` öffnen und Brand-Variablen setzen (Farben, Fonts, Inhalte)
3. `public/images/` mit Kundenmaterial füllen
4. `npm install && npm run dev`
5. Anpassen, deployen

## Status

Templates werden in **Phase 7** befüllt — nachdem alle vier Demos und das Portfolio fertig sind. Bis dahin sind die Ordner Platzhalter.
